d3.compose

d3.compose adds advanced chart layout and composition to d3 and d3.chart. Create rich, data-bound charts by composing charts (like Line and Bars) and components (like Axis, Title, and Legend).

d3.compose depends on the following libraries:

  • D3.js (>= 3.0.0)
  • d3.chart (>= 0.2.0)
  • Underscore.js (>= 1.5.0)

Introduction

When creating complex charts with D3.js and d3.chart, laying out and sizing parts of the chart are often manual processes and creating composable and reusable components becomes very difficult. d3.compose creates a standard base that charts and components are built on top of that automatically handles layout and sizing and can be composed and layered to created rich, data-bound charts.

Examples

Line

To get started with d3.compose, use d3.select('...').chart('Compose') and pass in a options function that takes in the data to be drawn and returns options for the charts and components to be drawn. The below example takes in series data, creates options for scales that are shared between charts and components, sets options for a Line chart, and sets options for axes and title components.

{% highlight js %} var example_1 = d3.select('#js-example-1').append('svg') .chart('Compose', function(data) { // Manipulate data as needed // Create options for scales shared between charts and/or components var scales = { x: {data: data, key: 'x'}, y: {data: data, key: 'y'} }; return { charts: { // Add a d3.chart('Line') chart // with given data and scales line: { type: 'Line', data: data, // Set options for chart xScale: scales.x, yScale: scales.y, interpolate: 'monotone' } }, components: { 'axis.x': { type: 'Axis', position: 'bottom', scale: scales.x }, 'axis.y': { type: 'Axis', position: 'left', scale: scales.y }, title: { type: 'Title', position: 'top', text: 'Line Chart' } } }; }) .margins({top: 0, right: 15, bottom: 5, left: 5}); example_1.draw([ { key: 'a', values: [ {x: 0, y: 10}, {x: 10, y: 20}, {x: 20, y: 50}, {x: 30, y: 100} ] }, { key: 'b', values: [ {x: 0, y: 50}, {x: 10, y: 45}, {x: 20, y: 15}, {x: 30, y: 10} ] } ]); {% endhighlight %}

XY Extension

Thanks to the flexibility of the options function, you can create extensions that wrap common behavior and reduce boilerplate for composing charts. The d3.compose.xy extension includes helpers/default values for axes, titles, and legends that make creating standard XY charts much simpler. Here's the Line example using the d3.compose.xy extension:

{% highlight js %} var example_2 = d3.select('#js-example-2').append('svg') .chart('Compose', function(data) { var scales = { x: {data: data, key: 'x'}, y: {data: data, key: 'y'} }; // Use xy extension for axes, title, and legend helpers return d3.compose.xy({ charts: { line: { // same as above } }, axes: { // Add x and y axes x: {scale: scales.x}, // defaults: type: Axis, position: bottom y: {scale: scales.y} // defaults: type: Axis, position: left }, // Add chart title title: 'Line Chart' }); }); // Draw chart with series data: [{key, values}, {key, values}, ...] example_2.draw([ // same as above ]); {% endhighlight %}

Line-Bar Values

The next example demonstrates how easy it is to compose are chart out of multiple child charts. Each chart layer is aligned and all components are laid out around the chart layer. This chart uses XYValues to center the value at each x-value/key/index and that needs to be specified for the axes with type: 'AxisValues'.

{% highlight js %} var example_2 = d3.select('#js-example-2').append('svg') .chart('Compose', function(data) { var input = data.input; var results = data.results; var scales = { x: {type: 'ordinal', data: input, key: 'x'}, y: {data: input, key: 'y'}, y2: {data: results, key: 'y', domain: [0, 100]} }; return d3.compose.xy({ charts: { input: { type: 'LineValues', data: input, xScale: scales.x, yScale: scales.y }, results: { type: 'Bars', data: results, xScale: scales.x, yScale: scales.y2 } }, axes: { x: {type: 'AxisValues', position: 'bottom', scale: scales.x}, y: {position: 'left', scale: scales.y, title: 'Input'}, y2: {position: 'right', scale: scales.y2, title: 'Results'} }, title: 'Input vs. Output' }); }); example_2.draw({ input: [ { key: 'input', values: [{x: 'a', y: 100}, {x: 'b', y: 200}, {x: 'c', y: 300}] } ], results: [ { key: 'run-1', values: [{x: 'a', y: 10}, {x: 'b', y: 30}, {x: 'c', y: 20}] }, { key: 'run-2', values: [{x: 'a', y: 15}, {x: 'b', y: 25}, {x: 'c', y: 20}] } ] }); {% endhighlight %}

Custom Chart

d3.compose makes it straightforward to create reusable charts with a common Chart base to extend from and helpers and mixins for common functionality.

{% highlight js %} var helpers = d3.chart.helpers; var mixins = d3.chart.mixins; d3.chart('Chart').extend('Dots', helpers.mixin(mixins.Series, mixins.XY, { initialize: function() { var base = this.base.append('g').attr('class', 'chart-dots'); // seriesLayer wraps series functionality // so that standard layer approach can be used this.seriesLayer('Dots', base, { dataBind: function(data) { return this.selectAll('circle') .data(data, this.chart().keyValue); }, insert: function() { return this.append('circle'); }, events: { merge: function() { var chart = this.chart(); this .attr('cx', chart.x) .attr('cy', chart.y) .attr('r', chart.r); } } }); }, // helpers.di binds chart to "di" functions // so that "this" refers to the element (as expected) r: helpers.di(function(chart, d, i) { return chart.rValue(); }), // helpers.property creates get/set property rValue: helpers.property('rValue', { default_value: 5 }) })); example_3 = d3.select('#js-example-3').append('svg') .chart('Compose', function(data) { var scales = { x: {data: data, key: 'x'}, y: {data: data, key: 'y'} }; return { charts: { line: {type: 'Line', data: data, xScale: scales.x, yScale: scales.y}, dots: {type: 'Dots', data: data, xScale: scales.x, yScale: scales.y} } }; }); example_3.draw([ { key: 'a', values: [{x: 0, y: 10}, {x: 10, y: 100}, {x: 50, y: 0}] }, { key: 'b', values: [{x: 50, y: 10}, {x: 30, y: 70}, {x: 0, y: 0}] } ]); {% endhighlight %}

Custom Component

In addition to custom charts, d3.compose allows you to make custom components with a common Component base that includes layout helpers.

{% highlight js %} d3.chart('Component').extend('Key', { initialize: function() { this.layer('Key', this.base, { dataBind: function(data) { return this.selectAll('text') .data(data); }, insert: function() { return this.append('text'); }, events: { merge: function() { this.text(this.chart().keyText) } } }) }, keyText: helpers.di(function(chart, d, i) { return d.abbr + ' = ' + d.value; }) }); var example_4 = d3.select('#js-example-4').append('svg') .chart('Compose', function(data) { var scales = { x: {type: 'ordinal', data: data.values, key: 'x'}, y: {domain: [0, 20]} } return d3.compose.xy({ charts: { bars: { type: 'Bars', data: data.values, xScale: scales.x, yScale: scales.y } }, axes: { x: {type: 'AxisValues', scale: scales.x} }, components: { key: {type: 'Key', data: data.key, position: 'right'} } }); }) .margins({top: 10, right: 10, bottom: 10, left: 10}); example_4.draw({ values: [{ key: 'a', values: [{x: 'js', y: 10}] }], key: [{abbr: 'js', value: 'javascript'}] }); {% endhighlight %}

Compose

d3.chart('Compose') allows charts and components to be composed into a rich charts. Using the options property, charts and components can be bound to data and customized with options to create intricate and dynamic charts.

{% highlight js %} d3.select('#chart') .chart('Compose', function(data) { // Compose charts, components, axes, legend, title into single chart // Work with given data var scales = { x: {data: data.input, key: 'x'}, y: {data: data.input, key: 'y'}, y2: {data: data.output, key: 'y'} }; return { charts: { // Create Line and Bars charts with given options input: { type: 'Line', data: data.input, xScale: scales.x, yScale: scales.y }, output: { type: 'Bars', data: data.output, xScale: scales.x, yScale: scales.y2 } }, components: { // Create axes with given options 'axis.x': { type: 'Axis', scale: scales.x, position: 'bottom' }, 'axis.y': { type: 'Axis', scale: scales.y, position: 'left' }, 'axis.y2': { type: 'Axis', scale: scales.y2, position: 'right' }, title: {text: 'Input vs. Output'} } }; }); {% endhighlight %}

optionschart.options([options])
Get/set the options the chart. Can be an Object or a Function that returns an Object. All properties listed below are available as options (charts, components, etc.) and the given values are passed to the matching property. options are generally set when creating chart with d3.select('...').chart('Compose', options). Example:

{% highlight js %} // Get chart.options(); // Set (Object) chart.options({ charts: {/*... */}, components: {/* ... */} }); // Set (Function, takes in data and returns object) chart.options(function(data) { // Handle data... return { charts: {/*... */}, components: {/* ... */} }; }); // Set directly from d3.chart creation d3.select('#chart') .chart('Compose', function(data) { // ... }); {% endhighlight %}

drawchart.draw([data])
Draw the chart with the given data. data is passed to options function and child charts and components.

chartschart.charts([values])
Chart options used for composing charts. Each chart should use a unique key so that updates are passed to the correct chart. The type option should be a registered d3.chart.

{% highlight js %} chart.charts({ input: {type: 'Line', /* options to pass to Line chart */}, output: {type: 'Bars', /* options to pass to Bars chart */} }); {% endhighlight %}

componentschart.components([values])
Component options used for composing and laying out components. Each component should use a unique key so that updates are passed to the correct component. The type option should be a registered d3.chart component and all other options are passed to the component.

{% highlight js %} chart.components({ xAxis: {type: 'Axis', /* options to pass to Axis */}, legend: {type: 'Legend', /* options to pass to Legend */} }); {% endhighlight %}

marginschart.margins([values])
Set the top, right, bottom, and left margin between the chart layers and the surrounding components.

{% highlight js %} chart.margins({top: 10, right: 0, bottom: 0, left: 10});{% endhighlight %}

widthchart.width([value])
Get or explicitly set the chart width.

heightchart.height([value])
Get or explicitly set the the chart height.

events on, off, and trigger can be used to work with chart events. Available events:

  • enter:mouse (container: {x, y}, chart: {x, y}) — when mouse enters chart
  • move:mouse (container: {x, y}, chart: {x, y}) — when mouse moves within chart
  • leave:mouse — when mouse leaves chart

Line

The Line chart (d3.chart('Line')) is an XY chart that can take single or series data.

{% highlight js %} // Use within d3.chart('Compose') d3.select('#chart') .chart('Compose', function(data) { return { charts: { a: { type: 'Line', data: ..., xScale: ..., yScale: ..., other options... } } }; }); {% endhighlight %}

dataline.data([values])
Array or Series of {x, y} data for line.

{% highlight js %} line.data([ {x: 0, y: 10}, /* ... */ ]); line.data([ {key: 'series-a', values: [{x, 0, y: 10}]}, {key: 'series-b', values: [{x: 0, y: -10}]} ]); {% endhighlight %}

labelsline.labels([options])
Add labels to chart with given options (Use true to add labels with default options).

{% highlight js %} // Default labels line.labels(true); // Labels with options line.labels({/* options to pass to Labels */}) {% endhighlight %}

transition
delay, duration, and ease properties are available for controlling the line transitions as data updates.

XY

Bars

databars.data([values])
Array or Series of Values data for bars.

{% highlight js %} bars.data([10, 20, 30]); bars.data([{y: 10}, /* ... */]); bars.data([ {key: 'series-a', values: [{key: 'a', y: 10}]}, {key: 'series-b', values: [{key: 'a', y: -10}]} ]); {% endhighlight %}

displayAdjacentbars.displayAdjacent([value])
For Series data, display bars at same index/key/x next to each other (rather than on top of). Default is true.

itemPaddingbars.itemPadding([value])
Adjust the padding (in percentage) between bars at different index/key/x.

{% highlight js %} bars.itemPadding(0.10); // 10% of bar width as padding between bars {% endhighlight %}

labelsbars.labels([options])
Add labels to chart with given options (Use true to add labels with default options).

{% highlight js %} // Default labels bars.labels(true); // Labels with options bars.labels({/* options to pass to Labels */}) {% endhighlight %}

transition
delay, duration, and ease properties are available for controlling the bar transitions as data updates.

Values

Labels

Display labels for XY data (can be used standalone or within chart).

{% highlight js %} // Use within d3.chart('Compose') d3.select('#chart') .chart('Compose', function(data) { return { charts: { // Enable labels within chart (true or options) a: {type: 'Line', ..., labels: true} // Add labels directly labels: {type: 'Labels', labels: ..., xScale: ..., yScale: ...} } }; }); {% endhighlight %}

labelsLabels.labels([values])
Explicitly set label values (otherwise, they are drawn from parent chart's data). Use label in data to explicitly set label, otherwise y-value with format is used.

{% highlight js %} Labels.labels([ {x: 0, y: 10, label: '10 items'} ]); Labels.labels([ {key: 'series-a', values: [{x: 0, y: 10}]}, {key: 'series-b', values: [{x: 0, y: -10}]} ]) {% endhighlight %}

formatLabels.format([value])
Format y-values using d3.format. Can use d3.format directly or string, which will be passed to d3.format.

{% highlight js %} // The following are equivalent labels.format('0,000'); labels.format(d3.format('0,000')); {% endhighlight %}

positionLabels.position([value])
How to position label relative to point. Valid values: "top", "right", "bottom", "left".

offsetLabels.offset([value])
Distance to offset label from point. Can be value or {x, y}.

paddingLabels.padding([value])
Padding between label text and background.

anchorLabels.anchor([value])
Horizontal text anchor for label (default set based on position). Valid values: "start", "middle", "end".

alignmentLabels.alignment([value])
Vertical alignment for label (default set based on position). Valid values: "top", "middle", "bottom".

handleCollisionsLabels.handleCollisions([value])
Handle collisions between labels of different series (default is true).

transition
delay, duration, and ease properties are available for controlling the label transitions as data updates.

Standard XY properties are available: xKey, yKey, xScale, yScale, xMin, xMax, yMin, yMax. See the XY Mixin for details.

Axis

Set scale for axis. If xScale and/or yScale are given, scale is set automatically based on position. Can be d3.scale instance or scale options.

How to position axis relative to chart. Valid values: "top", "right", "bottom", "left", "x0", "y0".

Used internally with x0 and y0 positioning, translation is used to set {x, y} distance from edge of chart.

Set where labels are positioned relative to axis (default set based on position). Valid values: "top", "right", "bottom", "left".

orientationaxis.orientation([value])
Set axis vertical/horizontal orientation (default set based on position). Valid values: "vertical", "horizontal".

axis
The following d3.axis properties are available: ticks, tickValues, tickSize, innerTickSize, outerTickSize, tickPadding, and tickFormat.

delay, duration, and ease properties are available for controlling the axis transitions as data updates.

Extensions

Extensions are used for defining options for d3.compose that wrap common behavior, reduce boilerplate, and provide helpers for composing charts and components.

d3.compose.xy

The d3.compose.xy extension includes helpers for creating XY charts that include a title, axes, and legend. It provides sensible defaults and can reduce the amount of boilerplate needed to create standard XY charts.

{% highlight js %} d3.select('#chart') .chart('Compose', function(data) { var scales = { x: {/* ... */}, y: {/* ... */}, y2: {/* ... */} }; return d3.compose.xy({ // charts and components as normal charts: {/* ... */}, components: {/* ... */}, // Axes: // type: 'Axis' // position: x -> bottom, y -> left, x2 -> top, y2 -> right axes: { x: {scale: scales.x, title: 'X Axis'}, y: {scale: scales.y, title: 'Y Axis'}, y2: {scale: scales.y2, title: 'Y2 Axis'} }, title: 'Chart Title' }); // Transforms to return { charts: {/* ... */}, components: { /* ... */ 'axis.x': {type: 'Axis', position: 'bottom', scale: scales.x}, 'axis.x.title': {type: 'Title', position: 'bottom', text: 'X Axis'}, 'axis.y': {type: 'Axis', position: 'left', scale: scales.y}, 'axis.y.title': {type: 'Title', position: 'left', text: 'Y Axis'}, 'axis.y2': {type: 'Axis', position: 'right', scale: scales.y2}, 'axis.y2.title': {type: 'Title', position: 'right', text: 'Y2 Axis'}, title: {type: 'Title', position: 'top', text: 'Chart Title'} } } }); {% endhighlight %}

helpers

helpers (exposed as d3.chart.helpers) includes general purpose helpers that are used throughout d3.compose. Includes convenience functions for creating charts (property, di, and mixin), helpful calculations (dimensions, max, and min), and other common behavior.

propertyhelpers.property(name, [options])
Used throughout d3.compose, property is used to create an get/set property. Created properties are called with no arguments for get and called with value for set.
Options:

  • get (stored) — (Optional) getter function
  • set (value, previous) — (Optional) setter function. To override stored value, return {override: ...}. To run code after the value has been stored, return {after: fn}
  • default_value — Set the default value or function that returns default value
  • type — Set as "Function" if stored value is a function that should not be evaluated on get

valueOrDefaulthelpers.valueOrDefault(value, default)
If given value is not undefined, return given value, otherwise return default.

{% highlight js %} helpers.valueOrDefault(undefined, 4); // -> 4 helpers.valueOrDefault(2, 4); // -> 2 helpers.valueOrDefault(null, 10); // -> null {% endhighlight %}

dimensionshelpers.dimensions(element|selector)
Return the width and height of the given element or selector.

translatehelpers.translate(x, y)
Get formatted translate transform (translate(x, y))

rotatehelpers.rotate(degress, center)
Get formatted rotate transform, with optional {x, y} center (rotate(degrees x,y))

isSeriesDatahelpers.isSeriesData(data)
Check if the given data is series data.

{% highlight js %} helpers.isSeriesData([1, 2, 3]); // -> false helpers.isSeriesData([{values: [1, 2, 3]}, {values: [4, 5, 6]}]); // -> true {% endhighlight %}

maxhelpers.max(data, getValue)
Find the max in the given data, using the getValue accessor to get the value at each point.

{% highlight js %} helpers.max([ {x: 0, y: 0}, {x: 5, y: 20}, {x: 10, y: 10} ], function(d, i) { return d.y; }); // -> 20 {% endhighlight %}

minhelpers.min(data, getValue)
Find the min in the given data, using the getValue accessor to get the value at each point.

{% highlight js %} helpers.min([ {x: 0, y: 0}, {x: 5, y: 20}, {x: 10, y: 10} ], function(d, i) { return d.x; }); // -> 0 {% endhighlight %}

createScalehelpers.createScale(options)
Create d3.scale from the given options. If a d3.scale instance is given, it is passed through.
Options:

  • type — (Default: linear) Available options: "linear", "time", and all d3.scale[type...]
  • ... — All scale options (e.g. range, domain, rangeBands, etc.)
  • data — Data for automatically setting domain (requires "key" option)
  • key — Key for getting value from data

stylehelpers.style(styles)
Convert object containing styles into inline style string.

{% highlight js %} helpers.style({fill: 'blue', 'stroke-width': '2px'}); // -> fill: blue; stroke-width: 2px {% endhighlight %}

dihelpers.di(callback)
Create a "di" function that gets parent chart injected. Inside "di" functions, d3 sets the element to "this" and there is no good way to access both the parent chart and the element.

{% highlight js %} x: di(function(chart, d, i) { // "this" is the element // to call other di function with di: var other = chart.otherDi.call(this, d, i); }); {% endhighlight %}

bindDihelpers.bindDi(di, chart)
Bind given di function (from helpers.di) to given chart.

bindAllDihelpers.bindAllDi(chart)
Bind all di functions attached to given chart (from prototype or instance).

getParentDatahelpers.getParentData(element)
Get parent data for given element. Useful for getting series from data point.

mixinhelpers.mixin()
Combine mixins / prototypes into single prototype for chart.

Series

The Series mixin includes helpers for handling series data.

seriesKey
"di" function for getting the series' key.

seriesValues
"di" function for getting the series' values.

seriesClass
"di" function for getting the series' class.

seriesIndex
"di" function for getting the series' index.

seriesCount
"di" function for getting the number of series.

seriesData
"di" function for getting the overall series data for child element.

itemStyle
"di" function that includes that combines the data item's style with the series and chart styles.

{% highlight js %} var data = [ { key: 'series-1', style: {fill: 'green', stroke: 'green'}, values: [ { x: 0, y: 10, style: {stroke: 'yellow'} } ] } ]; chart.style({fill: 'blue', 'opacity': 0.8}); // itemStyle -> 'opacity: 0.8; fill: green; stroke: yellow' {% endhighlight %}

XY

The XY mixin includes helpers for handling XY array and series data.

xKeyxKey([key])
Get/set the key used to get x-values from the data (default is "x").

yKeyyKey([key])
Get/set the key used to get y-values from the data (default is "y").

xScalexScale([scale|options])
Get/set the x-scale for the data. Can be d3.scale or options that are passed to helpers.createScale. Range for scale is set automatically from the chart dimensions with setXScaleRange

yScaleyScale([scale|options])
Get/set the y-scale for the data. Can be d3.scale or options that are passed to helpers.createScale. Range for scale is set automatically from the chart dimensions with setYScaleRange

xMinxMin([value])
Get/set the minimum x-value.

xMaxxMax([value])
Get/set the maximum x-value;

yMinyMin([value])
Get/set the minimum y-value.

yMaxyMax([value])
Get/set the maximum y-value.

x
"di" for getting x-value for charting (uses xValue to extract from data and passes through xScale for scaling).

y
"di" for getting y-value for charting (uses yValue to extract from data and passes through yScale for scaling).

x0
"di" for getting x-value at x = 0 for charting (uses xValue to extract from data and passes through xScale for scaling).

y0
"di" for getting y-value at y = 0 for charting (uses yValue to extract from data and passes through yScale for scaling).

xValue
"di" for extracting x-value from data (by xKey).

yValue
"di" for extracting y-value from data (by yKey).

keyValue
"di" for extracting key from data (uses "key" if present in data or xValue).

setXScaleRangesetXScaleRange(x_scale)
Used to set the x-scale range for default and defined xScale. By default, uses chart width, but can be overriden for custom functionality.

setYScaleRangesetYScaleRange(y_scale)
Used to set the y-scale range for default and defined yScale. By default, uses chart height, but can be overriden for custom functionality.

getDefaultXScalegetDefaultXScale()
If no xScale is given, this method is used to create a default x-scale. Useful to override default scale options to match the chart options (e.g. scale type: ordinal).

getDefaultYScalegetDefaultYScale()
If no yScale is given, this method is used to create a default y-scale.

XYValues

The XYValues mixin extends the XY mixin with additional helpers for handling XY values array and series data.

itemPaddingitemPadding([value])
Get/set the padding (in percentage) between items at different index/key/x.

displayAdjacentdisplayAdjacent([value])
By default, items at the same index/key/x are displayed with the same x-coordinate ("layered"). Enable displayAdjacent to display the items next to each other.

adjacentX
"di" for getting x-coordinate of adjacently placed item.

adjacentWidth
"di" for getting width of adjacently placed item.

layeredX
"di" for getting x-coordinate of layered item.

layeredWidth
"di" for getting width of layered item.

itemWidth
Get the width of the item (layeredWidth by default, adjacentWidth if displayAdjacent is enabled).

Labels

The Labels mixin includes helpers for adding labels to an XY chart.

attachLabelsattachLabels()
Call attachLabels in the chart's initialize method to add a labels layer.

labelslabels([options])
Get/set options that are passed to labels. Use type to set the type of labels chart to add ("Labels" is default).

Hover

The Hover mixin includes helpers for listening for and handling mouse events.

onMouseEnteronMouseEnter(position)
Called when mouse enters containing chart (position = {container: {x, y}, chart: {x, y}}).

onMouseMoveonMouseMove(position)
Called when mouse moves within containing chart (position = {container: {x, y}, chart: {x, y}}).

onMouseLeaveonMouseLeave()
Called when mouse leaves the containing chart.

XYHover

The XYHover mixin extends the Hover mixin with helpers specific to XY charts.

getPoint
"di" for getting the getting point information ({x, y, d, i, j}) for given element.

getPointsgetPoints()
Get all point information ({x, y, d, i j}) for chart's data.

getClosestPointsgetClosestPoints(position)
Get point information ({x, y, d, i j}) for point of each series that is closest to given position ({x, y}).
